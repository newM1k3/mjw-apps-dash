# PocketBase Schema — `users` Collection

The dashboard authenticates against the built-in PocketBase `users` collection. In addition to the default PocketBase auth fields (`id`, `email`, `password`, `created`, `updated`), the collection requires one custom field and supports two optional Stripe fields.

## Required custom fields

| Field | Type | Default | Allowed values | Notes |
|---|---|---|---|---|
| `tier` | Plain text | `free` | `free`, `pro`, `enterprise` | Controls which tools the user can launch. Missing, blank, or unrecognised values are normalised to `free` by `src/lib/user.ts`. |

### Valid `tier` values

| Value | Access |
|---|---|
| `free` | Free-tier tools only. |
| `pro` | Free + Pro tools. |
| `enterprise` | All 15 tools. |

Any value outside this set (including empty string, `null`, or a custom string like `"admin"`) is treated as `free`.

## Optional fields

| Field | Type | Notes |
|---|---|---|
| `stripe_customer_id` | Plain text | Populated when a user subscribes via Stripe. Not used by the dashboard directly. |
| `stripe_subscription_id` | Plain text | Populated when a user subscribes via Stripe. Not used by the dashboard directly. |

## Sample user record shape

This is the shape the dashboard reads from `pb.authStore.model` after a successful login:

```json
{
  "id": "RECORD_ID_15CHARS",
  "email": "operator@yourroom.com",
  "tier": "pro",
  "stripe_customer_id": "cus_xxxxxxxxxxxxxxxx",
  "stripe_subscription_id": "sub_xxxxxxxxxxxxxxxx",
  "created": "2024-01-15 09:30:00.000Z",
  "updated": "2024-03-20 14:22:00.000Z"
}
```

A free-tier user with no Stripe data:

```json
{
  "id": "RECORD_ID_15CHARS",
  "email": "newuser@yourroom.com",
  "tier": "free",
  "created": "2024-06-01 08:00:00.000Z",
  "updated": "2024-06-01 08:00:00.000Z"
}
```

## Tier normalisation

`src/lib/user.ts` exports `normalizeTier(tier: unknown): UserTier`. It is the single authoritative place that validates and falls back to `free`. The hook (`src/hooks/usePocketBase.ts`) calls it before surfacing the tier to any UI component, so no component ever receives an invalid tier string.

## How to add the `tier` field in PocketBase

1. Open the PocketBase admin UI.
2. Navigate to **Collections → users**.
3. Click **Edit collection**.
4. Add a new **Text** field named `tier`.
5. Set the default value to `free`.
6. Save.

## Setting a user's tier

Edit the user record directly in the PocketBase admin UI and set `tier` to `free`, `pro`, or `enterprise`.

For automated tier changes (e.g. after a Stripe webhook), use a PocketBase admin API call or a serverless function with the PocketBase service-role key to update the field — never from the client-side dashboard.

## Default PocketBase auth fields (read-only reference)

| Field | Type | Notes |
|---|---|---|
| `id` | Text (15 chars) | Auto-generated by PocketBase. |
| `email` | Email | Used as the login identifier. |
| `username` | Text | Auto-generated; not used by this dashboard. |
| `verified` | Boolean | Email verification status. Dashboard does not require verified email. |
| `created` | DateTime | Record creation timestamp. |
| `updated` | DateTime | Last update timestamp. |
